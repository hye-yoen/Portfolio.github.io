Hangle Project

소개

Hangle은 Spring Boot와 React로 개발된 머신러닝 대회 플랫폼입니다.

사용자는 각 대회에 참가하여 예측 결과 CSV를 제출하고, 자동 채점을 통해 리더보드에서 자신의 순위를 확인할 수 있습니다.

JWT 기반 인증과 권한 관리, REST API 구조를 통해 안정적인 데이터 처리와 클라이언트–서버 간 통신을 제공합니다.

배포 링크 : 나중에

GitHub : github 메인링크

타임라인

  • 2025.10.20 ~ : 프로젝트 기획 및 화면 설계
  • 2025.11 : Spring Boot + MySQL 백엔드 구축 + React 프론트엔드 개발
  • 2025.11 : Docker 기반 통합 테스트 및 배포

역할 (기여도)

리더보드 최고 점수 관리, 재제출 처리, 실시간 순위 재계산, 검색·필터링 기능 구현 (10%)

주요 기능 목록

구분 사용기술
Frontend React 18, Axios, React Router, MUI
Backend Spring Boot 3.x, JPA, Lombok, Spring Security, JWT
Database MySQL 8.x
DevOps Docker, Docker Compose, Nginx, AWS EC2
Tools IntelliJ, VSCode, GitHub, Postman

문제 해결

리더보드 검색 입력 시 결과가 변경되지 않는 문제가 발생했다. 조사 결과, 프론트엔드에서는 최초 로딩된 데이터만 상태에 저장해 검색 시 서버에 재요청하지 않고 정적 필터링을 시도하고 있었고, 검색어도 API 요청 URL에 전달되지 않는 구조였다.
이 문제를 해결하기 위해 검색 이벤트와 Axios 호출을 명확히 연결하고, 검색 시마다 최신 데이터를 서버에서 가져오는 구조로 변경했다. 그 결과 검색 기능이 정상적으로 동작했고 프론트–백엔드 간 데이터 전달 구조가 정리되었다.
개발 과정에서 프론트엔드 상태 관리와 API 연동 구조가 부족했다는 점을 깨달았고, 이후에는 기능 설계 단계에서 상태·이벤트·API 연동을 먼저 고려하는 방식으로 개선하였다.
